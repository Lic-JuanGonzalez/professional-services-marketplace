# Git workflow

- Branch names: `feat/`, `fix/`, `chore/`, `docs/` prefixes
- Commits: Conventional Commits format (`feat:`, `fix:`, `chore:`)
- Never commit to `main` directly — always branch + PR
- Never force-push `main` or `develop`
- Squash merge only — no merge commits on main
- PR title = commit subject that will land on main
